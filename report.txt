A. ABSTRACT (150-200 words)
================================================================================

Network security has become increasingly critical in modern computing environments 
where cyber threats continue to evolve in sophistication and frequency. This project 
presents a Real-Time Network Intrusion Detection System (IDS) designed to monitor, 
analyze, and detect malicious network activities through live packet capture and 
intelligent threat analysis.

The proposed solution implements a C++ backend (ids_integrated.cpp) utilizing the 
libpcap library for deep packet inspection, combined with a modern glassmorphism-styled 
web dashboard (dashboard.html) for real-time visualization. The system incorporates 
advanced detection mechanisms including SYN flood detection, port scan identification, 
similar packet burst analysis, and customizable rule-based threat classification.

Key features include live packet streaming, threat severity classification (Critical, 
High, Medium, Low, Info), IP monitoring capabilities, machine learning-inspired 
feedback mechanisms for reducing false positives, PCAP file replay functionality, 
and comprehensive analytics with visual charts. The expected contribution is a 
lightweight, efficient, and user-friendly IDS solution suitable for small to 
medium-sized network environments, providing security administrators with actionable 
insights for threat mitigation.

C. INTRODUCTION
================================================================================

BACKGROUND
----------
The exponential growth of internet-connected devices and digital infrastructure 
has created unprecedented opportunities for cyber attackers. Network intrusion 
detection systems serve as a critical line of defense, monitoring network traffic 
for suspicious patterns and potential security breaches. Traditional signature-based 
detection methods, while effective against known threats, struggle to identify 
novel attack vectors and zero-day exploits.

Modern IDS solutions require real-time processing capabilities, intuitive 
visualization interfaces, and adaptive learning mechanisms to keep pace with 
evolving threat landscapes. The integration of web-based dashboards has become 
essential for providing security analysts with immediate, accessible insights 
into network health and security status.

IMPORTANCE OF THE DOMAIN
------------------------
Network security represents a multi-billion dollar industry with annual growth 
rates exceeding 10%. Organizations face average breach costs of $4.45 million 
(IBM Security Report 2023), making proactive intrusion detection not merely a 
technical necessity but a business imperative. Key factors driving IDS importance:

1. Regulatory Compliance: GDPR, HIPAA, PCI-DSS mandate security monitoring
2. IoT Proliferation: Growing attack surface with connected devices
3. Remote Work: Distributed networks require enhanced monitoring
4. Ransomware Epidemic: Real-time detection critical for prevention
5. Data Protection: Intellectual property and personal data safeguarding

PROBLEM STATEMENT
-----------------
Current network security solutions face several critical challenges:

1. COMPLEXITY: Enterprise-grade IDS solutions like Snort or Suricata require 
   extensive configuration and specialized knowledge, making them inaccessible 
   for small to medium businesses.

2. COST: Commercial solutions carry significant licensing costs and hardware 
   requirements that exceed budget constraints for many organizations.

3. REAL-TIME VISIBILITY: Many existing tools lack intuitive, real-time 
   visualization capabilities, forcing analysts to parse through log files 
   manually.

4. FALSE POSITIVES: High false alarm rates lead to alert fatigue, causing 
   genuine threats to be overlooked or ignored.

5. ADAPTABILITY: Rigid rule-based systems fail to adapt to new attack 
   patterns without manual intervention.

PROPOSED SOLUTION
-----------------
This project proposes a lightweight, integrated Network IDS with the following 
core components:

1. BACKEND ENGINE (ids_integrated.cpp - 1026 lines)
   - Real-time packet capture using libpcap
   - Multi-threaded architecture for concurrent processing
   - Embedded HTTP server for API endpoints
   - Intelligent threat scoring with feedback learning
   - File-based persistence for rules and configurations

2. WEB DASHBOARD (dashboard.html - 1239 lines)
   - Modern glassmorphism UI design
   - Real-time event streaming
   - Interactive charts (Chart.js) for analytics
   - Filter controls for IP, severity, and status
   - Modal interfaces for configuration management

3. INTELLIGENCE FEATURES
   - Packet signature similarity detection
   - Threat multiplier based on user feedback
   - Dynamic severity adjustment
   - Historical pattern analysis

CRITICAL ANALYSIS
-----------------
The reviewed literature reveals a clear evolution from simple signature matching 
to sophisticated machine learning approaches. However, practical deployment gaps 
persist:

1. COMPLEXITY VS. USABILITY: Academic solutions prioritize detection accuracy 
   over operational simplicity, creating adoption barriers.

2. DATASET DEPENDENCY: Most ML-based approaches require extensive training 
   datasets that may not reflect real-world traffic patterns.

3. REAL-TIME CONSTRAINTS: Deep learning models often sacrifice processing 
   speed for detection accuracy, unsuitable for high-throughput networks.

4. FALSE POSITIVE MANAGEMENT: Few solutions incorporate feedback mechanisms 
   to learn from analyst corrections.

This project bridges these gaps by combining lightweight signature matching 
with behavioral analysis and user feedback learning, wrapped in an accessible 
web interface suitable for non-expert deployment.

WORKFLOW
--------

1. INITIALIZATION
   - Load rules.txt (detection patterns)
   - Load monitored_ips.txt (IP watchlist)
   - Load intelligence.dat (learned threat multipliers)
   - Start HTTP server on port 8080

2. USER INTERACTION
   - User opens browser to http://localhost:8080
   - Dashboard loads and begins polling API endpoints
   - User selects network interface and clicks "Start"

3. PACKET CAPTURE
   - Backend spawns capture thread
   - libpcap opens interface in promiscuous mode
   - Packets are parsed in packet_handler()

4. THREAT ANALYSIS
   - Each packet checked against:
     * SYN flood detection (>50 SYN from same IP)
     * Port scan detection (>15 unique ports)
     * Similar packet burst detection (>8 identical signatures)
     * Rule keyword matching
     * Monitored IP elevation
     * Intelligence multiplier adjustment

5. REAL-TIME DISPLAY
   - Events stored in captured_events vector
   - Dashboard polls /api/events every 500ms
   - Charts update with /api/analytics data
   - Filters applied client-side

6. USER FEEDBACK
   - User marks event as "True Threat" or "False Alarm"
   - Backend adjusts threat_multiplier for source IP
   - Intelligence persisted for future sessions

USE CASE 1: START PACKET CAPTURE
--------------------------------
Actor: Security Analyst
Preconditions:
  - IDS backend is running (./ids_integrated)
  - User has root/sudo privileges
  - Network interface is available

Main Flow:
  1. User opens dashboard in browser
  2. User clicks interface dropdown
  3. System displays available interfaces with IPs
  4. User selects target interface (e.g., eth0)
  5. User clicks "Start" button
  6. System spawns capture thread
  7. Status badge changes to "CAPTURING"

Postconditions:
  - Packets are being captured and analyzed
  - Event stream shows live traffic
  - Statistics begin incrementing

Alternative Flow:
  - If interface unavailable: Display error message
  - If already capturing: Show "already_running" status


USE CASE 2: MONITOR LIVE EVENTS
-------------------------------
Actor: Security Analyst
Preconditions:
  - Packet capture is active
  - Events have been detected

Main Flow:
  1. User views Live Event Stream section
  2. Events display with timestamp, IPs, description, severity
  3. User enters IP address in filter field
  4. System filters events matching IP
  5. User selects severity filter (e.g., "Critical")
  6. System shows only Critical events
  7. User hovers over event entry
  8. Feedback buttons appear (True Threat / False Alarm)

Postconditions:
  - User has visibility into network activity
  - Filtered view aids threat prioritization

Alternative Flow:
  - No events detected: Display "Monitoring..." placeholder
  - Connection lost: Status shows "Disconnected"


USE CASE 3: PROVIDE FEEDBACK ON ALERTS
--------------------------------------
Actor: Security Analyst
Preconditions:
  - At least one event is displayed
  - User has analyzed the event

Main Flow:
  1. User hovers over suspicious event
  2. "True Threat" and "False Alarm" buttons appear
  3. User determines event is a false positive
  4. User clicks "False Alarm" button
  5. System sends feedback to /api/feedback endpoint
  6. Backend reduces threat_multiplier for source IP
  7. Event is marked with grayscale styling
  8. False Alarms counter increments

Postconditions:
  - Intelligence updated for source IP
  - Future events from IP receive lower severity
  - intelligence.dat file updated

Alternative Flow:
  - If True Threat: Increase threat_multiplier
  - Event marked with red border indicator

APPENDIX E: SEVERITY LEVEL DEFINITIONS
--------------------------------------
CRITICAL - Immediate threat requiring urgent response
           (SYN flood, confirmed exploits)
HIGH     - Potentially dangerous activity warranting investigation
           (Port scans, similar packet bursts)
MEDIUM   - Suspicious behavior from monitored or unusual sources
           (Traffic from watchlist IPs)
LOW      - Noteworthy but low-risk events
           (Standard service access patterns)
INFO     - Normal traffic for baseline visibility
           (Routine connections)
